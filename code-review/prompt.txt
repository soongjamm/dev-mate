다음 HTML 템플릿을 받아서, 현재 디렉토리의 git diff를 기준으로 코드 리뷰를 채워 넣어 주세요.

코드리뷰 룰
1. **코드 의도 파악**: 의도를 명확히 묻고, 엣지 케이스나 버그 가능성을 지적해 주세요.
2. **테스트 커버리지**: 누락되거나 오래된 테스트가 있는지 확인해 주세요.
3. **코드 품질**: 가독성, 네이밍, 예외 처리, 보안, 성능 관점에서 검토해 주세요.
4. **PR 요약**: ‘Overall Review’ 섹션 아래에 주요 발견 사항과 실행 가능한 권장 사항을 요약해 주세요.
5. **prefix**: 각 코멘트에 적절한 prefix 를 붙여서 중요도나 목적을 알려주세요.

HTML 생성 룰
- HTML에서 “주요 변경 사항 요약”, “주요 위험 요소”, “권장 조치” 등 섹션 제목(heading)을 자동으로 찾아 해당 영역에 알맞은 불릿 리스트 형태의 리뷰 내용을 삽입합니다.
- “파일별 리뷰” 섹션에서는 각 파일마다 변경된 라인(추가·삭제)과 그에 대한 간단한 코멘트를 템플릿 내 파일 리뷰 구조(파일명 헤더, diff-line, comment-bubble 등)를 활용해 채워 넣습니다.
- HTML의 다른 구조나 스타일, 클래스명은 절대 수정하지 마세요.
- 모든 리뷰 코멘트와 리스트 항목은 한국어로 작성해 주세요.
- 이미 존재하는 파일이 있으면 덮어씁니다.
